The 0.5 release of the Java LAPACK is now available. This is an automatic translation from the original LAPACK 3.0 source code (Fortran 77) to Java. The current distribution consists of class files organized in the following packages:
  • org.netlib.blas - the BLAS routines
  • org.netlib.lapack - the double precision LAPACK routines
  • org.netlib.err - error reporting routine for LAPACK
  • org.netlib.util - utility routines and object wrappers
The source code is also available, but the bytecode transformer is required to recompile it into working bytecode (read this section of the tech report to understand why).
Download JLAPACK 0.5 as a gzipped tar file: jlapack-0_5.tgz

OR

We now have the JLAPACK 0.5 available as a ZIP file: jlapack-0_5.zip

To decompress the ZIP file, use your favorite unzip utility. You may need to specify that directories should be created rather than having everything unzipped into the current directory.

To decompress the .tgz file:

gunzip jlapack-0_5.tgz

tar xvf jlapack-0_5.tar

Or simply:

gunzip -c jlapack-0_5.tgz | tar xvf -

if you're into that sort of thing.

After downloading and uncompressing, follow the directions in the INSTALL file. Note especially the warnings against recompilation. Due to the postprocessing that we must perform on the class files to support GOTO statements, the code won't work when you recompile it unless you use the bytecode translator. The bytecode translator is part of the f2j source release and has currently been tested only under Solaris and Linux.

For those who would like a local copy of the javadoc-generated documentation for JLAPACK, we have it available in ZIP and tarred gzip formats:

jlapack-doc.zip

jlapack-doc.tgz

The BLAS/LAPACK testing routines are now available for download. The same recompilation warnings apply here, too. Download testers.tgz or testers.zip. As we test different on different machines and VMs, we will post a summary of the results. So far, the tests pass on:

Solaris 7 (sparc), Sun jdk 1.3.1, Hotspot

Solaris 7 (sparc), Sun jdk 1.2.2

Redhat 7.1 (x86), Sun jdk 1.3.1, Hotspot

Win2000 (x86), Sun jdk 1.2.2, classic vm, symcjit
