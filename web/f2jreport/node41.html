Block if

FORTRAN and Java share the same behavior for block if constructions, i.e. if-else-if statements, only one test may be successfully evaluated. The remaining are skipped, execution continues at the first statement after the last else-if block, or default else if present.

It appears from this code that each if, else if and else require a unique label. That is, a block if requires as many unique labels as exist if, else if and else statements in the block.
